# Session Module API Reference

`llm-session-whitebox` — zero-dependency white-box session manager for LLM agents.

## Install

```bash
pip install llm-session-whitebox           # core only (stdlib)
pip install llm-session-whitebox[viz]      # with visualization server
```

## Package structure

```
session/
├── models.py          # Typed dataclasses
├── session.py         # Core Session class
├── events.py          # EventBus pub/sub
├── tool_executor.py   # Tool execution with lifecycle
├── serializer.py      # JSON round-trip
├── checkpoint.py      # Versionable history + branching
├── _viz/server.py     # Flask visualization (optional)
└── tests/             # Test suite
```

## `session.Session`

Central white-box conversation manager. Stores typed messages, provides
structured views (turns, tool calls, stats), emits events.

### Constructor

```python
Session(session_id: str | None = None)
```

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `session_id` | `str \| None` | autogenerated | Unique session identifier |

### Message management

#### `add_user_message(content) → UserMessage`
Append a user message. Starts a new turn.

| Param | Type | Description |
|-------|------|-------------|
| `content` | `str \| list[dict]` | Plain text or structured content blocks |

#### `add_assistant_message(content, usage=None) → AssistantMessage`
Append an assistant message. Tool-call blocks are auto-indexed.

| Param | Type | Default | Description |
|-------|------|---------|-------------|
| `content` | `list[TextBlock \| ToolCallBlock]` | | Content blocks |
| `usage` | `dict[str, int] \| None` | `{}` | Token counts (`input_tokens`, `output_tokens`) |

#### `add_tool_result(tool_use_id, content, is_error=False) → ToolResult`
Record a tool execution result and link it to the originating ToolCallBlock.

#### `update_tool_call(tool_use_id, *, status, result, started_at, completed_at) → ToolCallBlock | None`
Update lifecycle fields and emit events.

#### `start_tool_call(tool_use_id) → ToolCallBlock | None`
Mark a tool as executing (convenience).

#### `complete_tool_call(tool_use_id, result) → ToolCallBlock | None`
Mark as completed or errored based on `result.is_error`.

### Query / white-box access

| Property / Method | Returns | Description |
|-------------------|---------|-------------|
| `messages` | `list[UserMessage \| AssistantMessage]` | Flat typed message list |
| `turns` | `list[Turn]` | Structured turn view (assembled from flat messages) |
| `tool_calls` | `list[ToolCallBlock]` | All tool-call blocks in order |
| `get_tool_call(tool_use_id)` | `ToolCallBlock \| None` | Lookup by tool_use_id |
| `get_tool_result(tool_use_id)` | `ToolResult \| None` | Lookup by tool_use_id |
| `last_message` | `UserMessage \| AssistantMessage \| None` | Most recent message |
| `last_assistant_text` | `str` | Text from most recent assistant message |
| `stats()` | `SessionStats` | Aggregate statistics |

### Stats (SessionStats)

| Field | Type | Description |
|-------|------|-------------|
| `total_turns` | `int` | Number of user→assistant cycles |
| `total_messages` | `int` | Total messages |
| `total_tool_calls` | `int` | Total tool invocations |
| `tool_calls_completed` | `int` | Successfully completed |
| `tool_calls_errored` | `int` | Failed with error |
| `total_input_tokens` | `int` | Cumulative input tokens |
| `total_output_tokens` | `int` | Cumulative output tokens |
| `total_duration_ms` | `int` | Sum of all turn durations |
| `avg_tool_latency_ms` | `float` | Average tool execution time |

### White-box editing

| Method | Returns | Description |
|--------|---------|-------------|
| `edit_message(message_id, new_content)` | `bool` | Edit any message's content |
| `delete_message(message_id)` | `int` | Remove a message + cascade cleanup |
| `retry_tool_call(tool_use_id)` | `bool` | Reset tool to pending for re-execution |
| `rollback_to(message_id)` | `int` | Truncate session to before this message |
| `inject_correction(text)` | `UserMessage` | Inject a correction message |

### Events

```python
session.events.on("message_added", lambda session, msg: ...)
session.events.on("tool_call_started", lambda session, block: ...)
session.events.on("tool_call_completed", lambda session, block: ...)
session.events.on("turn_completed", lambda session, turn: ...)
session.events.on("session_cleared", lambda session: ...)
```

### Compatibility

| Method | Description |
|--------|-------------|
| `to_dicts() → list[dict]` | Export as Engine-compatible `list[dict]` |
| `Session.from_dicts(messages, session_id) → Session` | Build from Engine-style dicts |
| `clear()` | Reset to empty state |

## `session.models`

All dataclasses live at `session.models`. Every object has `id`, `timestamp`, `meta`.

### Content blocks

```python
@dataclass
class TextBlock:
    text: str

@dataclass
class ToolCallBlock:
    tool_use_id: str
    name: str
    input: dict[str, Any]       # tool arguments
    status: str                  # pending | executing | completed | errored
    started_at: int | None       # epoch ms
    completed_at: int | None     # epoch ms
    result: ToolResult | None
    elapsed_ms: int | None       # computed property
    is_terminal: bool            # completed or errored

@dataclass
class ToolResult:
    tool_use_id: str
    content: str
    is_error: bool
```

### Messages

```python
@dataclass
class UserMessage:
    content: str | list[dict]
    text: str                    # computed property

@dataclass
class AssistantMessage:
    content: list[TextBlock | ToolCallBlock]
    text: str                    # computed (TextBlocks only)
    text_blocks: list[TextBlock] # filtered
    tool_calls: list[ToolCallBlock] # filtered
    has_tool_calls: bool
    usage: dict[str, int]        # token counts
```

### Turn & Stats

```python
@dataclass
class Turn:
    user_message: UserMessage
    assistant_messages: list[AssistantMessage]
    tool_results: list[ToolResult]
    all_tool_calls: list[ToolCallBlock]
    final_text: str
    elapsed_ms: int | None

@dataclass
class SessionStats:
    total_turns, total_messages, total_tool_calls: int
    tool_calls_completed, tool_calls_errored: int
    total_input_tokens, total_output_tokens: int
    total_duration_ms: int
    avg_tool_latency_ms: float
```

### Constants

```python
class Role:
    USER = "user"
    ASSISTANT = "assistant"

class ToolStatus:
    PENDING = "pending"
    EXECUTING = "executing"
    COMPLETED = "completed"
    ERRORED = "errored"
```

## `session.tool_executor.ToolExecutor`

Observable tool execution with full lifecycle tracking.

```python
class ToolExecutor:
    def __init__(self, tools: dict[str, Callable], session: Session): ...
    def register(name: str, fn: Callable) -> None: ...
    def execute(tool_call: ToolCallBlock, **extra_input) -> ToolResult: ...
    def execute_batch(tool_calls: list[ToolCallBlock], parallel: bool = False) -> list[ToolResult]: ...
```

`execute()` tracks status transitions, timing, and emits `tool_call_started` / `tool_call_completed` events. Errors are captured as results — never raises.

## `session.checkpoint.CheckpointStore`

Versionable history with git workspace tie-in. Storage: `~/.config/cc-mini/checkpoints/<session_id>/`.

```python
class CheckpointStore:
    def __init__(self, session_id: str, cwd: str | None = None): ...

    def save(session, label) -> SessionCheckpoint: ...    # snapshot + git info
    def list() -> list[SessionCheckpoint]: ...
    def get(checkpoint_id) -> SessionCheckpoint | None: ...
    def delete(checkpoint_id) -> bool: ...
    def restore(session, checkpoint_id) -> int: ...       # rollback to checkpoint
    def branches() -> list[str]: ...                      # list branch names
    def create_branch(session, checkpoint_id, branch_name) -> SessionCheckpoint: ...
    def switch_branch(session, branch_name) -> int: ...    # restore to branch tip
```

### SessionCheckpoint

```python
@dataclass
class SessionCheckpoint:
    id: str
    label: str                  # human name
    timestamp: int
    message_index: int          # position in session._messages
    message_count: int
    parent_checkpoint_id: str | None
    branch: str                 # "main" by default
    git_sha: str | None         # HEAD at checkpoint time
    git_branch: str | None
    git_dirty: bool             # uncommitted changes?
    git_files_changed: list[str]
```

## `session.events.EventBus`

```python
class EventBus:
    def on(event: str, callback) -> None: ...     # Register observer
    def off(event: str, callback) -> None: ...    # Remove observer
    def emit(event: str, *args, **kwargs) -> None: ...  # Fire event
    def clear() -> None: ...                      # Remove all
```

Callback errors are caught and printed to stderr — one broken observer
doesn't affect others.

## Visualization API

Start server: `session-viz` or `python -m session._viz.server` (port 8080).

| Method | Route | Description |
|--------|-------|-------------|
| `GET` | `/` | HTML session inspector |
| `GET` | `/api/session` | Full session state as JSON |
| `POST` | `/api/session/demo` | Load demo data |
| `POST` | `/api/session/message` | Add a message |
| `POST` | `/api/session/message/<id>/edit` | Edit message |
| `POST` | `/api/session/message/<id>/delete` | Delete message |
| `POST` | `/api/session/tool/<id>/retry` | Retry tool call |
| `POST` | `/api/session/rollback/<id>` | Rollback to message |
| `POST` | `/api/session/inject` | Inject correction |
| `POST` | `/api/session/checkpoint` | Create checkpoint |
| `GET` | `/api/session/checkpoints` | List checkpoints |
| `POST` | `/api/session/checkpoint/<id>/restore` | Restore checkpoint |
| `POST` | `/api/session/checkpoint/<id>/branch` | Branch from checkpoint |
| `DELETE` | `/api/session/checkpoint/<id>` | Delete checkpoint |
| `POST` | `/api/session/branch/<name>` | Switch branch |
